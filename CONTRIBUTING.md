# Contributing

Thanks for contributing!

## Guidelines

- Keep solutions **self-contained** (single source file per problem).
- Prefer **clear naming** and minimal global state.
- Add a short header comment at the top of each solution:
  - problem name
  - approach
  - time/space complexity
- Use **snake_case** filenames and avoid spaces.

## Adding a new solution

1. Pick the right folder:
   - `solutions/c/` for `.c`
   - `solutions/cpp/` for `.cpp`
2. Name the file in `snake_case`.
3. Ensure it compiles with warnings enabled.

## Commit messages

Use conventional-ish commits when possible:
- `feat: add <problem> solution`
- `fix: correct <problem> edge case`
- `docs: update README`
